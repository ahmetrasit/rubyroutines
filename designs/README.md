# Check-in UI Designs - "Soft Fade" Style

A collection of 20 interactive check-in UI designs optimized for both kids (5-12) and seniors (60+), using a stress-free "Soft Fade" color philosophy.

## üé® Design Philosophy: "Soft Fade"

Our designs celebrate task completion without creating pressure around incomplete tasks:

| State | Color | Hex | Purpose |
|-------|-------|-----|---------|
| **Incomplete** | Calm Blue-Gray | `#B0BEC5`, `#90A4AE` | Patient, no stress |
| **Complete** | Gentle Green | `#81C784`, `#66BB6A` | Satisfying relief |
| **Progress** | Encouraging Blue | `#90CAF9`, `#42A5F5` | Motivating support |
| **Backgrounds** | Soft Tints | `#F1F8E9`, `#E3F2FD` | Subtle celebration |

### Key Principles
- ‚úÖ **Equal visual weight** - Incomplete tasks aren't de-emphasized
- ‚úÖ **No red/yellow warnings** - Only calming colors
- ‚úÖ **Positive language** - "When ready", "At your pace"
- ‚úÖ **Relief over pressure** - Celebrates what's done, patient with what's not

## üéØ Design Specifications

### Typography
- **Dashboard:** 16-18px body, 24-26px headings
- **Kiosk:** 20-24px body, 36-42px headings
- **Font:** System fonts (-apple-system, BlinkMacSystemFont, 'Segoe UI')

### Touch Targets
- **Dashboard (Mobile):** Minimum 56px height
- **Kiosk (Tablet):** Minimum 120px height
- **Buttons:** 44px+ on mobile, 56px+ on tablet

### Spacing
- **Dashboard:** 16-24px padding, 12-16px gaps
- **Kiosk:** 24-32px padding, 16-24px gaps

### Accessibility
- **Contrast Ratio:** WCAG AA compliant (4.5:1 minimum)
- **Interactive Elements:** Clear focus states
- **Color:** Not the only indicator (shape + text + color)

## üîÑ Task Types

All designs support three task types:

### 1. Simple Tasks (Checklist)
- ‚úÖ One-time completion per reset period
- Examples: "Brush Teeth", "Make Bed"
- Visual: Single tap to toggle complete/incomplete

### 2. Multiple Check-in Tasks (Record Progress)
- üî¢ Up to 9 completions per task
- Examples: "Water" (drink water 8 times/day)
- Visual: Counter with + button

### 3. Progress Tasks (Record Progress)
- üìä Numeric value tracking with targets
- Examples: "Read Pages" (0/30 pages)
- Visual: Current/target display with + button

## üí° Design Decisions

### Why "Soft Fade"?
Traditional task UIs create stress by:
- ‚ùå Using red/yellow for incomplete tasks
- ‚ùå Making incomplete tasks look "wrong"
- ‚ùå Creating urgency where there shouldn't be

Our approach:
- ‚úÖ Incomplete = calm, patient blue-gray
- ‚úÖ Complete = gentle, satisfying green
- ‚úÖ Both states have equal visual weight
- ‚úÖ Positive, encouraging language throughout

### Why Two Families (A & C)?
- **A Family:** Better for fewer tasks (4-8), needs more clarity
- **C Family:** Better for many tasks (8+), space efficiency critical

### Why Dashboard vs Kiosk?
- **Dashboard:** Personal, mobile, portrait orientation
- **Kiosk:** Shared, tablet, landscape orientation, larger touch targets
